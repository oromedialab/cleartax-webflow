FAQ

Multi-ERP, multi-mandate — straight answers

The questions we get asked most by finance leads, ERP architects and CIOs scoping their EU e-invoicing project.

How does ClearTax integrate with my ERP, exactly?
It depends on which ERP you run — but the pattern is consistent: ClearTax uses your ERP's native extension model, so deployment looks like any other change in your existing release process.
  • SAP — ABAP add-on in the /CLEARTAX/* namespace, transported via STMS like any support pack
  • Oracle Fusion — REST API ingestion + FBDI bulk loads, deployed through Oracle Integration Cloud (OIC)
  • Oracle EBS — concurrent programs invoked from AR/AP open interfaces, with custom EXT-table writeback
  • NetSuite — managed SuiteApp listed on SuiteCloud, SuiteScript triggers on invoice creation
  • Dynamics 365 F&O — X++ extension deployed through LCS, with event handlers on invoice posting
  • Dynamics 365 Business Central — AppSource extension in AL, with page and table extensions
  • Sage X3 — 4GL extension on SINVOICE / PINVOICE, transported through standard Sage patches
  • Other ERPs — REST API push (JSON/XML), OData v4 endpoint, or iPaaS integration (MuleSoft, Boomi, CPI, Azure Logic Apps)
Whichever path, the result is the same: no core ERP modifications, no proprietary middleware, no impact on your ERP vendor support.
Which ERPs do you support?
60+ certified connectors, plus a global REST API for everything else. The top tier:
  • SAP — ECC 6.0 EHP7+, S/4HANA 1709–2023, S/4HANA Cloud (Private & Public Editions), Business One, ByDesign
  • Oracle — Fusion Cloud Financials, E-Business Suite R12, NetSuite, PeopleSoft, JD Edwards
  • Microsoft Dynamics — 365 Finance & Operations, 365 Business Central
  • Sage — X3, 100, 100c, 50
  • Mid-market & vertical — Infor M3, Infor LN, Workday Financials, Odoo, Cegid, Unit4, Epicor, IFS, SYSPRO, QAD, Acumatica, Priority, Tally, Zoho Books
Don't see yours? The global REST API works with any system that can emit invoice data — including custom in-house ERPs and home-grown billing platforms.
We run multiple ERPs across subsidiaries. Can ClearTax consolidate them?
Yes — this is one of the most common patterns we deploy. A typical multi-ERP setup: SAP S/4HANA at HQ, NetSuite at acquired subsidiaries, Sage in a regional finance shared-service centre. Each entity uses its own connector, but everything funnels into the same ClearTax Compliance Cloud, with the same unified cockpit, the same audit trail, the same compliance reporting.

Your tax team sees one consolidated view of pending submissions, failed validations, and status by entity — regardless of which ERP issued the invoice. Auditors get one trail. CFOs get one dashboard. Most importantly: you don't license a separate compliance vendor for each ERP.
How does this compare to my ERP's native compliance tooling?
Every major ERP vendor has built some flavour of native compliance tooling — SAP has Document & Reporting Compliance (DRC), Oracle has Tax Reporting Cloud, Microsoft has Electronic Reporting (ER), Sage bundles a France PA. They work well in narrow scenarios. ClearTax is the alternative when:
  • You need mandates the native tool doesn't yet cover (or covers with significant limitations)
  • You want one platform for all geographies — not separate country licences from each ERP vendor
  • You operate multiple ERPs and need a single compliance posture across them
  • You need confirmed PA accreditation in France — most ERP-native tools are not Plateformes Agréées
  • You want a faster go-live than the ERP vendor's own integration partner typically delivers
ClearTax also coexists fine with native tools — run native for one country, ClearTax for the rest. We see this often.
Which EU mandates are covered?
Every major EU mandate, live or imminent:
  • France — DGFiP, mandatory September 2026. ClearTax is a confirmed Plateforme Agréée (PA)
  • Belgium — Peppol BIS 3.0, B2B mandate live January 2026
  • Poland — KSeF, mandatory February 2026 (large) / April 2026 (SME)
  • Germany — Phased rollout 2025–2028 (receive obligation already in force, issuance phased by turnover)
  • Spain — Verifactu live for invoice systems, B2B mandate approved (technical regs pending)
  • Italy — SDI (live since 2019), FatturaPA format
  • Romania — RO e-Factura (CTC model, live since July 2024)

New mandates are added through configuration in the Compliance Cloud — not as new development projects in your ERP.
What about Germany's phased rollout and Spain's Verifactu vs B2B regime?
Germany phases the e-invoicing obligation by turnover and date:
  • Jan 2025 — all VAT-registered businesses must be able to receive structured e-invoices (EN 16931 / XRechnung / ZUGFeRD)
  • Jan 2027 — issuance becomes mandatory for businesses with >€800K annual turnover
  • Jan 2028 — issuance becomes mandatory for all businesses
ClearTax supports both XRechnung and ZUGFeRD formats, and routes via Peppol for cross-border flows.

Spain runs two parallel regimes:
  • Verifactu (the invoice-billing-system requirements under Royal Decree 1007/2023) — already live, voluntary in 2026, mandatory phased thereafter
  • B2B mandate — approved under the Crea y Crece law, awaiting final technical regulations (expected 2026/27)
ClearTax covers both — including the regional TicketBAI regimes in the Basque Country and Navarre, plus FacturaE for B2G.
What's the architecture between our ERP and the ClearTax cloud?
Outbound (ERP → ClearTax): the ERP-native connector triggers on invoice creation and pushes a structured payload over an authenticated HTTPS connection to the ClearTax Compliance Cloud. Real-time for high-priority flows, batch for bulk runs.

ClearTax cloud: converts to the target format (UBL 2.1 / CII / Factur-X / FA(2) / FatturaPA / XRechnung / ZUGFeRD / Peppol BIS), validates against the relevant schema and business rules, submits to the tax authority (DGFiP / KSeF / SDI / ANAF / BMF / AEAT / Peppol AP), and persists the lifecycle status.

Inbound (ClearTax → ERP): status writeback through the same authenticated channel, updating ERP-specific fields (Z-tables in SAP, EXT tables in Oracle EBS, custom records in NetSuite, table extensions in Business Central).

Hosting: ClearTax runs in the geographic region your data residency requirements demand. For EU customers, the EU AWS regions.
How is data residency and security handled for EU customers?
ClearTax is ISO 27001 and SOC 2 Type II certified. For EU customers, all production data stays in EU AWS regions and never crosses the boundary. Transit is TLS 1.3, at rest is AES-256. GDPR-compliant data handling throughout — DPA available, sub-processor list published, right-to-erasure workflows in place. Authentication is OAuth 2.0 with rotating tokens, optionally hardened with mTLS for high-sensitivity deployments. Role-based access in the cockpit maps to your ERP's existing role model (SAP PFCG, Oracle roles, Dynamics security roles, NetSuite roles).

What happens when the mandate changes — does that need a new ERP project?
No. Mandate changes are handled in the ClearTax cloud, not in your ERP. When DGFiP publishes External Specs v3.2, when KSeF adds a validation rule, when Germany finalises its phase-2 schemas — ClearTax updates the format engine and validation library, and your ERP connector continues sending the same structured payload it always did. The translation layer absorbs the change. Your ERP team rarely needs to ship a new release. This is the core architectural value of keeping mandate-specific complexity outside your ERP.
Can we test against sandbox tax authorities before going live?
Yes — and we strongly recommend it. ClearTax's DEV and QA environments connect to the official sandbox endpoints for every supported mandate (DGFiP PiSte, KSeF demo, ANAF test, BMF test bed, AEAT pre-production, Peppol test corner). You can run your real ERP invoices through the full pipeline before flipping production. This is also where AI-mapped fields get validated against real schema responses.
